웹 퍼블리싱 문서 가이드

HTML 개요

HTML은 HyperText Markup Language의 약자로, 웹 문서의 구조와 의미를 정의하는 마크업 언어입니다.

HTML은 로직을 처리하지 않으며, 콘텐츠의 의미를 정확히 표현하는 것이 핵심 목적입니다.

HTML의 역할

HTML은 제목, 문단, 목록, 링크, 이미지 등 콘텐츠의 구조를 명확히 구분합니다.

시맨틱 마크업

시맨틱 태그를 사용하면 접근성과 SEO 품질이 크게 향상됩니다.

대표적인 시맨틱 태그

header, main, section, article, footer 등이 시맨틱 구조를 구성합니다.

CSS 개요

CSS는 HTML로 작성된 구조에 시각적 스타일을 입히는 역할을 합니다.

CSS의 기본 구성

선택자, 속성, 값으로 구성되며 선언 방식에 따라 우선순위가 결정됩니다.

레이아웃 시스템

Flexbox와 Grid는 현대 웹 레이아웃의 핵심 도구입니다.

Flexbox

1차원 레이아웃에 적합하며 정렬과 간격 제어에 강점이 있습니다.

Grid

2차원 레이아웃에 최적화되어 있으며 복잡한 UI 구성에 유리합니다.

JavaScript 개요

JavaScript는 웹 페이지에 동적인 동작과 로직을 부여합니다.

DOM 조작

DOM API를 통해 HTML 요소를 선택하고 이벤트 및 상태를 제어할 수 있습니다.

모듈 시스템

ES Module을 사용하면 코드 구조를 명확하게 분리할 수 있습니다.

모듈의 장점

유지보수성, 재사용성, 가독성이 크게 향상됩니다.

문서형 UI 설계

문서형 UI는 정보 전달과 탐색이 가장 중요한 서비스 유형입니다.

목차(TOC)의 역할

TOC는 문서 구조를 빠르게 파악하고 원하는 위치로 이동할 수 있게 도와줍니다.

검색 UX

검색 기능은 긴 문서에서 사용자 피로도를 크게 줄여줍니다.

키보드 기반 탐색

단축키를 활용한 검색과 이동은 고급 사용자 경험을 제공합니다.
